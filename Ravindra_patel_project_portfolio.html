Search
Problem

Approach

Tracked CTR, search exit rate, and lead conversions post-launch

↑Improved filter adoption and click-through rates across 500+ categories
↑Higher lead conversions by aligning filters with real buyer intent
↓Reduced filter clutter — eliminated irrelevant, redundant, and city-based filters

Approach

bathrok wiperbathrock wiperbathroom wiper

Search Quality context: Google's Search Quality Evaluator Guidelines (Section 19.0) establish that for obviously misspelled queries, raters must evaluate results based on user intent — not literal spelling. A search for "bathrok wiper" should be treated as "bathroom wiper." This audit ensured IndiaMART's ML models met the same standard: correctly resolving intent even when queries arrive imperfect. Poor spell correction translates directly to a "Fails to Meet" result in quality frameworks — the very failures this audit identified and helped fix.

04
Agent Next Action Classification (AI Training – E-commerce Support)
Freelance · Firstsource — Training data annotation for AI customer support decision models
AI / Annotation
Objective
Trained AI models to correctly predict what a customer support agent should do next in an e-commerce conversation — whether to ask for more information, start troubleshooting, provide solution options, verify identity, transfer to a specialized team, or close the conversation.
What I Did
Wrote realistic multi-turn conversations between a customer and support agent for diverse e-commerce scenarios
Created the final customer message (Input) that the AI model must analyze to decide the next action
Wrote step-by-step reasoning traces explaining why a particular next action is the only logical outcome
Applied metadata rules for each scenario: tone (formal/informal), cultural context, product category, and complexity level
Ensured ground-truth actions (e.g., Verify Identity, Troubleshooting, Solution Options) were unambiguous given the conversation context
Example Scenario
Customer asks agent to check their order details → correct next action is Verify Identity, because accessing account information requires authentication first. The reasoning trace must explain this chain of logic explicitly so the model learns the decision boundary.
Impact
↑Improved model's ability to understand conversation context and detect customer intent
↑Helped the AI learn to follow proper support workflows reliably across diverse scenarios
↑Strengthened model decision consistency through structured, unambiguous annotation methodology

export.indiamart.com · International buyers — missing company-name search preventing direct supplier discovery
Search / Discovery
Issue Identified

Aggregated 5 buyer tickets into a structured case study; recommended a seller contact opt-out setting and enquiry-consent enforcement
